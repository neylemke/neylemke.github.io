Installing Jekyll

Posted on Jan 08, 2017 By Ney Lemke

Jekyll is an wonderful tool that allows us to generate static html pages that can be used at github. The entire process seems easier if you follow the github pages protocol. But I found it rather difficult to follow.

The process that I suggest is the following:

  • Create a directory for your blog blogdir
  • Choose a Theme for Jekyll themes
  • Install the theme under blogdir
  • Create a project at GitHub called username.github.io
  • Clone the page using one of the prefined themes or not under blogdir (this dir is called githubdir)
  • Install ruby (jekyll is based on ruby)
  • Install Jekyll
sudo gem install jekyll
sudo gem install jekyll-paginate
  • Install dependencies for your theme
  • Configure the theme on the local filesystem
  • Buid your page using your theme 
    jekyll build
  • Generate a page or a blog post
  • Start the server 
    jekyll serve --watch
  • You can edit your page and keep the server running, since changes will cause a new buid
  • Test the page using the browser http://127.0.0.1:4000/
  • Build the page using jekyll setting the output to the github cloned page 
    jekyll build -d githubdir
  • Commit and push your changes 
    git add -all
git commit -m "description"
git push

    Now go to your Blog Page at username.github.io .
